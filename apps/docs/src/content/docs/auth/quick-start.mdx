## üöÄ 5-Minute Setup

### Step 1: Start the Development Servers (1 minute)

```bash
# Terminal 1: Start Convex backend
cd packages/backend
npx convex dev

# Terminal 2: Start web application  
cd apps/web
npm run dev

# Or use the monorepo command:
pnpm run dev
```

### Step 2: Test Password Authentication (1 minute)

1. Open [http://localhost:3003](http://localhost:3003)
2. Click **"Continue with Email"**
3. Create an account or sign in
4. ‚úÖ **Done!** You now have working authentication

### Step 3: Set Up OAuth (Optional - 3 minutes)

If you want Google and GitHub sign-in:

1. **Google OAuth**:
   - Go to [Google Cloud Console](https://console.cloud.google.com/)
   - Create OAuth 2.0 credentials
   - Add redirect URI: `http://localhost:3003/auth/google/callback`
   - Copy Client ID and Secret to `packages/backend/.env.local`

2. **GitHub OAuth**:
   - Go to [GitHub Developer Settings](https://github.com/settings/developers)
   - Create new OAuth App
   - Add callback URL: `http://localhost:3003/auth/github/callback`
   - Copy Client ID and Secret to `packages/backend/.env.local`

## üîß Required Convex Environment Variables (set via CLI)

Convex auth reads OAuth credentials from the Convex environment (not .env files). For local dev, set these on your local deployment:

```bash
cd packages/backend
npx convex env set SITE_URL "http://localhost:3003"
npx convex env set AUTH_GOOGLE_ID "<your-google-client-id>.apps.googleusercontent.com"
npx convex env set AUTH_GOOGLE_SECRET "<your-google-client-secret>"
# Optional if using GitHub OAuth
npx convex env set AUTH_GITHUB_ID "<your-github-client-id>"
npx convex env set AUTH_GITHUB_SECRET "<your-github-client-secret>"
```

Verify with:

```bash
npx convex env get SITE_URL
npx convex env get AUTH_GOOGLE_ID
npx convex env get AUTH_GOOGLE_SECRET
```

### `apps/web/.env.local`
```bash
# Convex Configuration
VITE_CONVEX_URL=http://127.0.0.1:3210
```

Note: The monorepo `pnpm dev` script fails fast if `SITE_URL`, `AUTH_GOOGLE_ID`, or `AUTH_GOOGLE_SECRET` are missing. You can bypass the check if you only need password auth by running:

```bash
PULSE_DEV_SKIP_AUTH_CHECK=1 pnpm dev
```

## üÜò Need Help?

If something isn't working:

1. **Check the logs** in both terminal windows
2. **Verify environment variables** are set correctly
3. **Clear browser data** for localhost:3003
4. **Restart both servers** if needed

The authentication system is designed to work out of the box - if you're having issues, it's likely a simple configuration problem that can be quickly resolved.
